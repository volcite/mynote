# 高成約率LP作成スキル

このスキルは、ユーザーのビジネスに基づき「売れるLP（ランディングページ）」を作成します。

## 使用方法

1. Phase 1: ヒアリング（商品・ターゲット・強み・文体の収集）
2. Phase 2: Webリサーチ（生の声・競合調査）
3. Phase 3: コンセプト設計（常識の否定・ポジショニング）
4. Phase 4: 文体分析（スタイルのインストール）
5. Phase 5: 構成案の作成
6. Phase 6: LP原稿の執筆
7. ファイル保存

# ★★★ 知識ベース（Reference Knowledge） ★★★

以下の知識を全ての出力に適用し、成約に導くLPに仕上げてください。

## 1. ★大原則★ 4つのNOT

顧客はあなたのLPに対してこの4つの壁を持っている。各フェーズで突破せよ。

| NOT | 壁 | 突破法 |
|-----|-----|--------|
| **Not Open** | 開かない | FV（ファーストビュー）で衝撃を与える。スクロールさせる力 |
| **Not Read** | 読まない | 見出しとリズムで引き込む。太字だけ読めば意味が通じるようにする |
| **Not Believe** | 信じない | 証拠・実績・権威性・お客様の声を積み上げる |
| **Not Act** | 動かない | オファーの魅力・限定性・緊急性で背中を押す |

## 2. ★最重要★ PASONA法則（全体の構成骨子）

LPの流れはこの法則で構成する。

```
1. Problem（問題提起）
   └─ ズバリ悩みを言い当てる。「こんなお悩みありませんか？」は陳腐。本音を突く。

2. Affinity（親近感）
   └─ 痛みに共感し、近づく。「私もかつては同じでした」（V字ストーリー）

3. Solution（解決策）
   └─ 独自の解決アプローチを提示。「なぜ今まではダメだったのか」を説明する。

4. Offer（提案）
   └─ 具体的な商品内容・価格・保証を提示。Value Stackで価値を積み上げる。

5. Narrowing（絞り込み）
   └─ 限定性・緊急性。「誰でも買える」はNG。「本気の人だけ」と絞る。

6. Action（行動）
   └─ 背中を押す。CTAボタンを複数配置。「今すぐ申し込む」
```

## 3. ベネフィットの3層構造

「時間が節約できます」「お金が稼げます」で止まるな。深掘りせよ。

```
第1層（機能的ベネフィット）
├─ 何ができるようになるか？（例：自動化ツールが使える）
│
第2層（感情的ベネフィット）
├─ その結果、どんな感情になるか？（例：毎朝の憂鬱から解放される）
│
第3層（自己実現ベネフィット）
└─ 周りの目はどう変わるか？人生はどう変わるか？（例：家族から尊敬される、理想の自分になれる）
```

**★必ず第3層まで掘り下げる★**

## 4. コンセプト設計（パズル思考）

競合と同じことを言うな。「常識の否定」で差別化せよ。

- **世間の常識**: 「〇〇するには△△が必要」
- **今回の主張**: 「実は△△は不要。□□だけでいい」

### 「ずらし」のテクニック

同じ「手段」「ベネフィット」でも、ターゲットをずらすと新しい市場が生まれる。

- 例：「不動産投資」→ ターゲットを「サラリーマン→週末だけ働きたい人」にずらす
- 例：「ただの水」→ ターゲットを「健康志向の人→エナジードリンク代わりを求める若者」にずらす（Liquid Death）

### 「目新しい」ではなく「真新しい」

全く新しいものを作る必要はない。既存の要素の「組み合わせ」や「切り口」を変えるだけで新しく感じさせられる。

## 5. ブランディングの鉄則（イメージ戦略）

### ★最重要★ マイナスを作らない

- ブランドイメージが崩れるのは一瞬。修復には多大な時間がかかる。
- プラスを積み上げるよりも、マイナス要素を生まないことが先決。
- **マイナス要素（弱み・欠点）があるなら、先に自分から開示せよ。** 後から発覚すると信頼が崩壊する。

### イメージは発信側が決める

- ブランドは「顧客が決めるもの」ではない。「どう思われたいか」を発信側が主体的に決め、それを広げる活動。
- LPの文体・デザイン・言葉選びは、すべて「持たせたいイメージ」から逆算する。

## 6. 価格提示の心理学

### アンカリング効果

最初に高い価格（定価、比較対象）を見せてから、実際の価格を見せると安く感じる。

```markdown
通常価格: ~~98,000円~~
特別価格: **49,800円**（50%OFF）
```

### Value Stack（価値の積み上げ）

本体商品だけでなく、特典を積み上げて「これだけついてこの価格！？」と思わせる。

```markdown
【含まれるもの】
✅ 本編（10万円相当）
✅ 特典1: テンプレート集（3万円相当）
✅ 特典2: 質問し放題サポート（5万円相当）
───────────────
合計18万円相当 → 49,800円
```

### リスクリバーサル（リスクゼロ化）

- 返金保証、無期限サポート、成果保証など。
- 「買わない理由」をすべて潰す。

# ★★★ AIがやりがちな失敗と対策（Absolute Rules） ★★★

## 失敗1: 「きれいな文章」を書いてしまう

- **現象**: 教科書的な、面白みのない、誰にでも当てはまる文章になる。
- **本質**: 嫌われないように配慮しすぎている。
- **対策**: ★「特定の誰か」に強烈に刺さる言葉（隠語・スラング・心の叫び）を使う★
- **意識**: 8割の人にスルーされても、ターゲットの1人が泣いて喜べば勝ち。

## 失敗2: ベネフィットが浅い

- **現象**: 「時間が節約できます」「お金が稼げます」で止まる。
- **本質**: 表面的な機能しか見ていない。
- **対策**: ★「ベネフィットの3層構造」を深掘りする★
- **具体化**: その結果、生活の何が変わる？どんな感情になる？周りの目はどう変わる？

## 失敗3: 競合と同じことを言う（クリシェ）

- **現象**: 「誰でも簡単」「最強メソッド」など、使い古された言葉を使う。
- **本質**: 思考停止してパターンを使っている。
- **対策**: ★「常識の否定」を入れる★
- **差別化**: 競合が「A」と言っているなら、こちらは「実はBだ」と別の角度から切る。

## 失敗4: 文体サンプルを見ていない（重要）

- **現象**: サンプルをもらっても、結局いつものAI口調（綺麗で硬い敬語）に戻る。
- **本質**: サンプルを「参考程度」にしか見ていない。
- **対策**: ★サンプルを分析し、その特徴（リズム・語尾・改行）を強制的に適用する★
- **禁止**: サンプルと異なる一人称や語尾を使うことは厳禁。

## 禁止表現

- **AI特有の硬い表現**: 「効率化」「最適化」「包括的な」「網羅的な」「体系的な」「〜することが可能です」
- **カタカナ語の乱用**: 「ソリューション」「パラダイムシフト」（ターゲットによるが基本NG）
- **弱気な表現**: 「〜かもしれません」「〜だと思います」「いかがでしたか？」
- **上から目線**: 説教臭い語り口「〜すべきです」
- **嘘の権威性**: 架空の数字や実績は絶対NG

# 実行プロセス

## Phase 1: ヒアリング

ユーザーに以下の質問を行ってください。一度に全ての質問を提示し、回答を求めてください。

```markdown
## LP作成ヒアリングシート

### 1. 商品とゴール

**【商品・サービス内容】**
何を売りますか？（詳細に）

**【ゴール（CV）】**
読者に何をしてほしいですか？（購入、問い合わせ、LINE登録など）

**【価格】**
定価、キャンペーン価格、松竹梅など

---

### 2. ターゲット（ペルソナ）

**【誰に売りますか？】**
年齢、職業、今の悩み、生活スタイルなど。

**【その人が抱えている「地獄」は？】**
「このまま放置したらいつか訪れる最悪の未来」は？
（例：誰にも必要とされず閑職に追いやられる、借金が膨らむ、等）

---

### 3. 強みと実績（権威性）

**【あなたの武器は？】**
- 具体的な実績（数字）
- 独自のメソッドや哲学
- 他社には絶対真似できない要素
- 開発秘話やストーリー

---

### 4. 文体サンプル（★重要）

**【あなたの文章を再現します】**
あなたの文体（リズム、語尾、一人称）を再現するため、
過去のNote記事、ブログ、SNS投稿などをコピペで貼り付けてください。（500文字以上推奨）

※特に指定がない場合は、プロのコピーライターとして「熱量高め・論理的」なトーンで執筆します。
```

---

## Phase 2: Webリサーチ（AIが実行）

ヒアリング後、**必ずWeb検索ツールを使用して**以下を調査する。

```
【検索アクション 1：ターゲットの憑依】
検索クエリ例：
・「[テーマ] つらい」「[テーマ] 辞めたい」「[テーマ] 失敗」
・Yahoo!知恵袋、5ch、Twitter等の「本音」を探す。

★収集するもの：
・ターゲットが実際に使っている「嘆きの言葉」「隠語」
・具体的すぎる失敗エピソード（リアリティの確保）

【検索アクション 2：競合の分析】
検索クエリ例：
・「[テーマ] 教材」「[テーマ] スクール」「[テーマ] コンサル」

★収集するもの：
・競合が使い倒している「手垢のついたフレーズ（クリシェ）」
・（これを避けるために調査する）
```

---

## Phase 3: コンセプト設計（AIが提案）

リサーチ結果を元に、「売れる切り口」を提案する。

```markdown
## LPコンセプト定義

1. **コンセプト名（仮）**: [名称]
2. **キャッチコピー案**:
   - 案1: [コピー]
   - 案2: [コピー]
   - 案3: [コピー]
3. **★「常識の否定」ポイント**:
   - 世間の常識：「〇〇するには△△が必要」
   - 今回の主張：「実は△△は不要。□□だけでいい」
4. **ターゲットの「インサイト（隠れた本音）」**: [本音]
5. **持たせたいブランドイメージ**: [イメージ（例：親しみやすい専門家、信頼できる兄貴分）]
```

※ここでユーザーの合意を得る。

---

## Phase 4: 文体分析（AIが実行）

ユーザーから提供されたサンプルテキスト（質問4の回答）を分析し、以下の特徴を抽出して箇条書きにする。
**以降の執筆は、必ずこの分析結果に従って行うこと。**

```markdown
## 文体分析結果

1. **一人称**: 私 / 僕 / 俺 / 自分
2. **語尾の傾向**: 〜です/ます / 〜だ/である / 体言止め / 問いかけ
3. **リズムと改行**: 短文が多いか / 長文か / 改行の頻度
4. **特徴的な口癖や言い回し**: [具体例]
5. **感情の出し方**: 冷静 / 熱血 / 論理的 / エモーショナル
```

※サンプルがない場合は、「信頼感のあるプロフェッショナルなトーン」と定義する。

---

## Phase 5: 構成案（骨子）の作成

PASONA法則に基づいたLPの全体構成案（見出しレベル）を作成し、提示する。ユーザーの合意を得てから執筆に進む。

### LP構成テンプレート

```markdown
## LP構成案

### 1. ファーストビュー（Problem / Hook）
*   キャッチコピー
*   サブキャッチ
*   CTA（1回目）

### 2. 痛みの共感と深掘り（Affinity）
*   「こんな悩みありませんか？」（陳腐にならないよう本音を突く）
*   放置した場合の「地獄の未来」

### 3. 希望の提示と自己紹介（Solution / Authority）
*   「私もかつては同じでした」（V字ストーリー）
*   解決策の概要
*   実績・権威性の提示

### 4. 常識の否定 / 独自メソッド
*   なぜ今ままでの方法はダメだったのか？
*   本当の原因は〇〇だった

### 5. 商品・サービスの詳細（Offer）
*   サービス内容
*   特典（Value Stack）
*   こんな人に向いている / 向いていない

### 6. お客様の声 / 実績（Social Proof）

### 7. 価格の提示（Offer続き）
*   アンカリング（比較対象の提示）
*   価格開示
*   保証（リスクリバーサル）
*   CTA（2回目）

### 8. よくある質問（FAQ）

### 9. 最後のメッセージ（Action / Closing）
*   背中を押すメッセージ
*   CTA（3回目）
```

---

## Phase 6: LP原稿の執筆

合意した構成に基づき、Markdown形式で執筆する。

### ★重要：出力形式

*   **「そのままコピー＆ペーストすればLPとして完成するMarkdown」**を出力する。
*   余計な挨拶や解説（「ここでは〜を意識しました」等）は、**LPの本文中には絶対に入れない**。
*   ファイル作成ツールを使用し、必ずファイル（.md）として保存する。

### 執筆時の注意点（Markdown装飾）

*   **見出し（#）**: 適切に階層化する。
*   **強調（**太字**）**: スキャンして読む人（飛ばし読み）のために、**太字だけ読めば意味が通じる**ようにする。
*   **リスト**: 箇条書きを多用し、読みやすくする。
*   **余白**: 段落は短くし、スマホでの可読性を意識する。

---

## Phase 7: 保存と納品

1. 執筆した内容を**一つの完全なMarkdownファイル**として `09_Draft` フォルダに保存する。
2. 保存するファイルの中身は、**LPの原稿のみ**とする。
    *   ❌ AIのコメント「以下がLPです」「ここは工夫しました」
    *   ❌ 思考プロセス
    *   ⭕ LPのタイトル、見出し、本文のみ
3. ユーザーには保存先のパスを提示し、「こちらをそのままコピペしてご使用ください」と伝える。

-   **保存先フォルダ**: `09_Draft`
-   **ファイル名**: `LP_[テーマ]_[YYYYMMDD].md`（例: `LP_AI自動化コンサル_20260209.md`）

# 最終チェック（LP Check）

出力前に以下を自問してください。

-   **4つのNOTを突破しているか？**: 開かない、読まない、信じない、動かない、それぞれに対策があるか？
-   **ベネフィットは第3層まで掘り下げたか？**: 機能→感情→自己実現まで深掘りしているか？
-   **常識の否定は入っているか？**: 競合と同じことを言っていないか？
-   **文体サンプルに忠実か？**: 一人称、語尾、リズムは分析結果通りか？
-   **マイナス要素を先出ししたか？**: 弱みや注意点を隠していないか？
-   **CTAは十分にあるか？**: 最低3箇所はCTAボタン（申し込みリンク）があるか？
