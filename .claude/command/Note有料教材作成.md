# 有料教材コンテンツ生成スキル（NBT）

このスキルは、note/Brain/Tips向けの「売れて満足度が高い」有料教材コンテンツを生成します。

## 使用方法

1. Phase 1: ヒアリング（7つの質問）
2. Phase 2: テーマ分析（生の声リサーチ・常識の否定特定）
3. Phase 3: 文体分析
4. Phase 4: スマホ最適化を意識した構成設計
5. Phase 5: 無料部分の執筆（LP機能）
6. Phase 6: 有料部分の執筆
7. Phase 7: 特典の設計（口コミ誘発型）
8. Phase 8: 画像挿入箇所リスト出力
9. Phase 9: Markdownファイル出力
10. Phase 10: 追加装飾リスト出力

---

# ★★★ 知識ベース（Reference Knowledge） ★★★

## 1. 購買心理フレームワーク

### PASONA法則（無料部分の構成骨子）

| ステップ | 役割 |
|---------|------|
| **Problem** | 問題提起：ズバリ悩みを言い当てる |
| **Affinity** | 親近感・共感：痛みに近づく |
| **Solution** | 解決策の提示 |
| **Offer** | 提案：具体的な商品内容 |
| **Narrowing** | 絞り込み・限定性 |
| **Action** | 行動喚起 |

### 4つのNOT（各パートで突破すべき壁）

| NOT | 壁 | 突破法 |
|-----|-----|--------|
| **Not Read** | 読まない | フックで突破 |
| **Not Believe** | 信じない | 証拠・実績で突破 |
| **Not Act** | 行動しない | 限定性・緊急性で突破 |
| **Not Understand** | 理解しない | シンプルな説明で突破 |

## 2. 心理学トリガー

- **希少性の原理**: 限定数、期間限定、今だけ
- **社会的証明**: お客様の声、実績数
- **権威性**: 専門家としての実績、経験年数
- **返報性の原理**: 無料で価値を先に与える
- **損失回避バイアス**: 「失うもの」を強調
- **アンカリング効果**: 高い価格を先に見せる
- **ツァイガルニク効果**: 未完了のものが気になる（有料ライン前で使用）
- **認知的不協和**: 常識を崩して揺さぶる

## 3. コピーライティングテクニック

### 常識の否定（★最重要★）

```
売れる有料教材には、必ずこの要素がある：

「見込み客が目標達成のために"本来は必要"だと思っていたこと、
"常識的にやらないといけない"と思っていたことを、
やらなくても目標達成できる」という構造。

【例】
├─ 常識：稼ぐにはスキルを身につけて時間をかける必要がある
└─ 否定：スキル不要、1日30分で収益化
```

### ベネフィットの3層構造

```
第1層（機能的）: 何ができるか
第2層（感情的）: どんな気持ちになれるか
第3層（社会的）: 周囲からどう見られるか
```

### 解像度の基準

```
【ダメな例（抽象的）】
「AI情報が多すぎて何をすればいいかわからない」

【良い例（生の声ベース・超具体的）】
「毎日AIインフルエンサーがプロンプト配布とかやってて参加するんだけど
実際使ってみると現場で使えないものばかりで毎度がっかりさせられてる」

→ 読んだ人が「これ俺のことじゃん」と思うか？
```

# ★★★ AIがやりがちな失敗と対策 ★★★

## 失敗1: 指示の表面だけ拾って本質を外す

- **現象**: 「緩急をつけて」と言われたら短文だけ増やす
- **対策**: 指示の「意図・目的」を理解してから実行する

## 失敗2: 参照すべきサンプルを見ていない

- **現象**: 文体サンプルを渡しても、その特徴を拾わない
- **対策**: ★サンプルを分析して特徴を箇条書きにしてから書き始める★

## 失敗3: パターン化する

- **現象**: 装飾の位置、文の長さ、語尾、構成が規則的になる
- **対策**: ★意図的に不規則にする★
- **禁止事項**:
  - 同じ装飾を連続して使わない
  - 同じ文の長さを3回以上連続させない
  - 「ポイントは3つあります」を毎回使わない
  - 箇条書きを毎回3つにしない（2つ、4つ、5つも使う）

## 失敗4: 目に見える要素だけ真似る

- **現象**: 一人称・語尾は真似るが、思考の流れ・間の取り方は真似ない
- **対策**: ★深い要素も言語化して再現する★
  - どこで断言し、どこで留保するか
  - どこで読者に問いかけるか
  - どこで具体例を出すか
  - どこで感情を出すか

## 禁止表現

- ❌ 「〜することが可能です」→「〜できます」に
- ❌ 「〜において」→「〜で」「〜では」に
- ❌ 「包括的な」「網羅的な」「体系的な」を使う
- ❌ 「最適化」「効率化」「最大化」を連発する
- ❌ 句読点を多用する（改行で区切る）
- ❌ 全ての文を同じ長さにする

# 実行プロセス

## Phase 1: ヒアリング

ユーザーに以下の質問を行ってください。

```markdown
## 有料教材作成ヒアリングシート

### 1. テーマと概要

**【テーマ】**
どんなテーマの有料教材を作りますか？
（例：「AI副業の始め方」「インスタ集客の方法」）

**【一言で言うと】**
この教材を一言で説明すると、何ができるようになる教材ですか？
（例：「完全初心者が30日でAI副業で月5万円を稼げるようになる」）

---

### 2. プラットフォーム

どのプラットフォームで販売しますか？

□ note
□ Brain  
□ Tips
□ 複数（指定してください）

---

### 3. ターゲット

**【年齢層・属性】**
（例：30代の会社員、主婦、フリーランス志望の20代など）

**【今の状態】**
その人は今、どんな状態で、何に困っていますか？

**【理想の状態】**
この教材を読んだ後、どうなっていてほしいですか？

---

### 4. あなたの実体験（★最重要・嘘厳禁）

**【なぜこのテーマを語れるのか】**
実績、経験年数、資格、学んできたことなど

**【失敗経験】**
（例：「最初の3ヶ月は全く稼げなかった」）

**【成功経験・実績】**
具体的な成功体験や実績。数字があればなお良い。

**【ターゲットへの思い】**
なぜこの教材を作ろうと思ったのですか？

---

### 5. 文体サンプル（★重要）

過去に書いたブログ記事やNote、SNS投稿をコピペで貼り付けてください。（500字以上推奨）

---

### 6. 特典と口コミ誘導

**【どのSNSで口コミを書いてもらいますか？】**
□ X（Twitter）の引用RT
□ Instagramのストーリーズ
□ その他

**【先着制限】**
□ あり → 先着何名？
□ なし

**【すでに用意している特典があれば教えてください】**
（なければ提案します）

---

### 7. 価格帯

□ 低価格帯（〜3,000円）
□ 中価格帯（3,000〜10,000円）
□ 高価格帯（10,000円〜）
```

---

## Phase 2: テーマ分析

### 生の声リサーチ（★必ずWeb検索を実行★）

```
【リサーチ先】
├─ Yahoo!知恵袋
├─ 5ch
├─ X（Twitter）
├─ YouTubeのコメント欄
└─ Google検索「〇〇 つらい」「〇〇 できない」

【検索クエリ例】
├─ 「[テーマ] 挫折 site:chiebukuro.yahoo.co.jp」
├─ 「[テーマ] できない site:twitter.com」
└─ 「[テーマ] やってみた 結果」

【収集すべき「生の声」】
├─ 具体的な失敗体験（数字、期間、金額入り）
├─ 感情の吐露（イライラ、焦り、自己嫌悪）
└─ コメント欄での共感の連鎖
```

### 常識の否定を特定

テーマが決まったら、必ず「常識の否定」を特定する。これが無料部分の核になる。

## Phase 3: 文体分析

ユーザーのサンプル文章を受け取ったら、以下を分析して箇条書きで出力する。

```
【表層的な要素】
├─ 一人称：私 / 僕 / 俺 / 自分
├─ 語尾パターン：です・ます / だ・である / 〜だよね
├─ 口癖・頻出表現
├─ 絵文字・記号の使用
└─ 専門用語の扱い

【深層的な要素】★これが重要★
├─ どこで断言するか
├─ どこで留保するか
├─ どこで読者に問いかけるか
├─ どこで具体例を出すか
├─ どこで感情を出すか
└─ 何を言わないか

【文章のリズム】
├─ 句読点の頻度（多い→硬い、少ない→人間らしい）
└─ 短文と長文がランダムに混在しているか
```

## Phase 4-6: 構成設計と執筆

### 無料部分の構成（LP機能）

1. フック（最初の3行）
2. 問題提起・共感
3. よくある間違い・失敗パターン（常識の否定）
4. 解決策の予告 + 超具体的ベネフィット
5. 購入しない未来（ペイン）
6. 著者の自己紹介・信頼性
7. この教材で得られること（目次）
8. お客様の声
9. 特典紹介
10. 行動喚起（CTA）→ 有料ライン

### 有料部分の各章構造

```
1. 章の冒頭
├─ この章で何を学ぶか（1文）
├─ なぜこれが重要か（Why）
└─ 読み終わった後の状態

2. 本文
├─ 概念の説明（What）
├─ 具体的なやり方（How）
├─ 具体例・事例（実体験ベース）
├─ よくある失敗と対処法
└─ ポイントの整理

3. 章の締め
├─ この章のまとめ（3行以内）
└─ 【今日やること】（1つだけ、5分以内でできること）
```

---

## Phase 7: 特典の設計（口コミ誘発型）

```
特典 = 口コミを誘発するための戦略的ツール

【実践を加速する系】★最も効果的
├─ そのまま使えるテンプレート
├─ コピペで使える文章集
└─ 自動化ツール（GPTs、スプレッドシートなど）

【個別化・サポート系】★高価値
├─ 著者への質問権（1回）
├─ 個別相談・ロードマップ作成
└─ 添削・フィードバック

【配置】
├─ 有料部分の冒頭：口コミを書くタスクを認識させる
└─ 有料部分の最後：詳しい受け取り方法を再掲
```

# 最終チェック

- **生の声リサーチ**: 知恵袋・SNS等で「見られてた？」レベルの具体性があるか？
- **常識の否定**: 認知的不協和を起こす構造になっているか？
- **文体再現**: サンプルの一人称・語尾・リズムを再現しているか？
- **パターン化回避**: 同じ装飾・同じ文の長さの連続を避けているか？
