# 自己紹介記事作成スキル

このスキルは、ユーザーの人生ストーリーをヒアリングし、「読者の心に響く自己紹介記事」を作成します。

## 使用方法

1. Phase 1: 情報収集（ヒアリング）
2. Phase 2: 初稿作成
3. Phase 3: ブラッシュアップ
4. Phase 4: 最終チェック・印象分析
5. ファイル保存

---

# ★★★ 知識ベース（Reference Knowledge） ★★★

以下の知識を全ての出力に適用し、「この人に会ってみたい」と思わせる記事に仕上げてください。

## 1. 自己紹介記事の本質

- **目的**: 読者に「この人は信頼できる」「この人から学びたい」と思わせること。
- **記事の役割**: 単なる経歴紹介ではなく、「なぜ今この仕事をしているのか」を伝える価値観の表明。
- **読者の期待**: 華やかな成功談ではなく、共感できる苦労と成長のストーリー。

## 2. ヒーローズジャーニー（V字回復の構造）

読者が引き込まれるストーリーは「V字回復」の構造を持っている。

```
     平穏な日常
         ↓
    【転落・どん底】
         ↓
     気づき・出会い
         ↓
    【復活・現在】
```

- **どん底から始める**: 冒頭で一番のどん底シーンを見せる（映画的オープニング）
- **そこから過去に戻る**: 「ここに至るまでには…」と時系列で語る
- **どん底と現在の価値観を繋げる**: 「〇〇を経験したからこそ、△△を大事にしている」

## 3. ピアノコピーテンプレートの活用

最強のフック（冒頭）の型：

```
私が〇〇すると、周囲が笑った。
でも、△△した瞬間……
```

自己紹介記事版：
```
借金○○万。手の震えが止まらない夜がありました。
でも今、私は……
```

## 4. 振り子の法則（マイナスの開示）

- **完璧を演じない**: 成功談だけでなく、失敗談・弱み・サボっていた時期を正直に書く。
- **後から出ると逆効果**: 弱みは先に開示する。後からバレると信頼崩壊。先に言うと信頼になる。
- **カッコつけない表現**: 「サボってた」「自業自得」「正直ダメだった」

## 5. 3Not原則の突破（自己紹介記事版）

- **見ない（Not Seen）**: どん底シーンの冒頭フックで目を引く
- **読まない（Not Read）**: 見出しを「中身が知りたくなる見出し」に変更
- **信じない（Not Believed）**: 弱みの開示と具体的なエピソードで信頼を獲得

## 6. 伏線と回収

- **序盤で性格や特徴を提示**: 「ベースは陰キャ」「人見知り」など
- **後半で回収**: 「だからこそ目の前の人に丁寧に向き合うスタイルが合っている」

# ★★★ AIがやりがちな失敗と対策（Absolute Rules） ★★★

## 1. 思考・文体の失敗対策

- **美化しすぎない**: 「苦労を乗り越えた素晴らしい私」という語り口はNG
- **カッコつけない**: 説教臭くならない。上から目線にならない。
- **感情を具体的に**: 「辛かった」→「手の震えが止まらなかった」「頭が真っ白になった」
- **パターンの排除**: 同じ文末、同じ長さの文章を連続させない

## 2. 禁止表現リスト

- **AI特有の硬い表現**: 「成長させていただきました」「学びを得ました」「〜において」
- **抽象的すぎる表現**: 「色々あった」「大変だった」「頑張った」→ 具体的に書く
- **キレイすぎる言い回し**: ポエムになってはいけない。泥臭さを残す。

# 実行プロセス

## Phase 1: 情報収集

まず以下の項目をユーザーに質問してください。一括で質問して構いません。
十分な情報が得られない場合は、掘り下げ質問を行ってください。

### 1-1. 基本情報
- 出身地・育った場所
- 家族構成（両親の特徴、育てられ方）
- 幼少期の性格（大人しい/活発、真面目/やんちゃ、など）

### 1-2. 学生時代のエピソード
- 小学校：印象的な出来事、部活/習い事、生徒会など
- 中学校：部活、先生との関係、転機となった出来事
- 高校：うまくいったこと/いかなかったこと、進路の悩み
- 大学：受験結果、バイト、サークル、海外経験など

### 1-3. 社会人としての経験
- 就職活動の経験（成功/失敗）
- 最初の仕事
- 転機となった出来事
- 失敗談・挫折経験（借金、詐欺、失恋、病気など）
- どん底からどう立ち直ったか

### 1-4. 現在と価値観
- 今やっている仕事・活動
- 大事にしている価値観（なぜそれを大事にするようになったか）
- これからの目標

### 1-5. 性格・人柄
- 自分の性格（陽キャ/陰キャ、リーダータイプ/サポートタイプ）
- コンプレックスや悩み
- モチベーションの源泉

### 掘り下げ質問（情報が足りない場合）
- 「その時、どんな気持ちでしたか？」
- 「なぜそれを大事にするようになったんですか？」
- 「その経験から何を学びましたか？」
- 「それは今の仕事/価値観にどう繋がっていますか？」

---

## Phase 2: 初稿作成

ヒアリングした情報をもとに、以下の構成で記事を作成してください。

### 構成テンプレート

#### 冒頭（フック）
- **どん底のシーンから始める**（映画的オープニング）
- 例：「借金○○万。手の震えが止まらない夜がありました。」
- その後「でも、ここに至るまでには〜」と過去に戻る

#### 本文（時系列）
1. 幼少期〜小学校
2. 中学時代
3. 高校時代
4. 大学時代
5. 社会人〜転機
6. どん底の経験
7. 立ち直り〜現在

#### 価値観セクション
- **どん底の経験と価値観を繋げる**
- 「〇〇を経験したからこそ、△△を大事にしている」という構成
- 例：「詐欺に遭ったからこそ分かった。楽して稼げる話なんてない。」

#### 締め
- これからの目標
- 読者への呼びかけ（仲間募集、話を聞かせてほしい、など）

### 文章のトーン

#### 文体
- 短文中心（1文は40文字以内を目安）
- 頻繁に改行
- 「ですます調」でカジュアルに
- 太字（**）で重要なフレーズを強調

#### 自己開示のバランス
- 成功談だけでなく**失敗談・弱みも正直に書く**
- 「サボってた」「自業自得」など、カッコつけない表現
- でも卑屈にならない

#### 感情の描写
- 転換点では「その時どう感じたか」を具体的に書く
- 例：「泣きそうになった」「頭が真っ白になった」「手の震えが止まらなかった」

---

## Phase 3: ブラッシュアップ

初稿完成後、以下の観点で改善してください。

### 3-1. 見出しの魅力化
すべての見出しを「中身が知りたくなる見出し」に変更：

| Before | After |
|--------|-------|
| 高校時代 | 人生で一番ダメだった高校3年間 |
| 大学受験 | 「まぁそうなるよな」自業自得の受験失敗 |
| AIとの出会い | 「これしかない」どん底で見つけた可能性 |

### 3-2. 転換点のドラマ性強化
人生が変わった瞬間に「その時の感情」を追加する。

### 3-3. 伏線と回収
序盤で触れた性格や価値観を、後半で回収する。
- 例：「ベースは陰キャ」→「だからこそ目の前の人に丁寧に向き合うスタイルが合っている」

### 3-4. どん底と価値観の接続
「〇〇を経験したからこそ、△△を大事にしている」という繋がりを明確にする。

### 3-5. 締めの強化
読者への具体的な呼びかけを追加する。

---

## Phase 4: 最終チェック

### チェックリスト
- [ ] 冒頭で読者を引き込めているか
- [ ] 時系列が分かりやすいか
- [ ] 失敗談・弱みが正直に書かれているか
- [ ] 転換点で感情が描かれているか
- [ ] どん底と価値観の繋がりが明確か
- [ ] 見出しが魅力的か
- [ ] 伏線が回収されているか
- [ ] 読者への呼びかけがあるか

### 印象分析
記事を読んだ人がどんな印象を持つか分析し、ネガティブな印象があれば修正を提案する。

---

## Phase 5: タイトルの作成

以下のポイントを押さえたタイトルを3つ提案してください。

### タイトルのポイント
- インパクトのある数字や出来事を入れる（例：「借金350万」「ホームレス」）
- 「〇〇した僕が今、△△していること」の構成
- 長すぎない（SNSで見切れない長さ）
- 【自己紹介】などのタグを付けてもOK

### 例
- 【自己紹介】詐欺で350万騙され、彼女と別れ、ホームレスも経験した僕が今、大切にしていること
- 手の震えが止まらなかった夜。借金350万、ホームレスになった僕の「その後」

## Phase 6: 記事の保存

作成した自己紹介記事を以下のルールで保存してください。

-   **保存先フォルダ**: `07_NotePost`
-   **ファイル名**: `YYYY-MM-DD_自己紹介_名前.md`（例: `2026-01-25_自己紹介_田中.md`）

# 最終チェック（Self-Introduction Check）

出力前に以下を自問してください。

-   **V字回復の構造になっているか？**: どん底→気づき→現在の流れが明確か？
-   **弱みが開示されているか？**: カッコつけすぎていないか？
-   **見出しは魅力的か？**: 「中身が知りたくなる見出し」になっているか？
-   **感情が具体的か？**: 「辛かった」ではなく「手が震えた」のように具体化されているか？
-   **読者への呼びかけがあるか？**: 「仲間募集」「話を聞かせてほしい」などのCTAがあるか？

# 開始

**「自己紹介記事作成を開始します。まずはあなたのことを教えてください。」** と出力し、Phase 1のヒアリング質問を行って開始してください。
