このスキルは、ユーザーのテーマに基づき「3日間限定セール価格で販売し、売上を最大化する有料Note記事」と「ローンチ用X投稿（5本）」を作成します。

## 使用方法

1. ヒアリングで要件を明確化
2. 価格正当化とバリュースタッキング設計
3. 構成案（プロット）の確認
4. 有料Note記事の作成・保存
5. X投稿5本の作成・保存
6. n8n Webhook送信
7. 公式LINE告知文の作成

# ★★★ 知識ベース（Reference Knowledge） ★★★

以下の知識を全ての出力に適用し、「お金を払ってでも読みたい」コンテンツに仕上げてください。

## 1. 有料コンテンツの鉄則

- **情報の解像度**: 無料情報は「What（何をするか）」まで。有料情報は「How（具体的にどうやるか）」と「Template（埋めるだけでできる）」を提供する。
- **価格の正当化（アンカー効果）**: 「本来ならスクールで30万円で教えている内容だが…」のように、定価の価値を高めてから割引価格を提示する。
- **満足度設計**: 読了後に「値段の10倍の価値があった」と思わせるため、理論だけでなく即実践できる「ワーク」や「チェックリスト」を含める。

## 2. コンセプト設計の鉄則（パズル思考）

- **コンセプトはパズル**: 真新しいもの（クリエイティビティ）ではなく、既存の要素を掛け合わせた「目新しいパッケージ」を作る。
- **パッケージを変える**: 商品の中身は変えなくても、見せ方・切り口を変えてターゲットを変える。
- **常識の破壊**: コンセプトの核は「ターゲットが信じている間違った手段の否定」にある。
    - ❌ 常識：「集客にはSNSが必要」
    - ✅ 破壊：「実はSNS不要で集客できる」
- **ターゲットの「ずらし」**: 同じ手段・同じベネフィットでも、お金を持っている別のターゲットにずらせないかを常に考える。
    - 例: 農家向け → 同じ手段でお金を持っている経営者向け
    - 例: ネットワーク洗脳解除 → 辞めさせたい社員を持つ社長向け
- **「何それ？」を生み出す**: 良いコンセプトは、初手で説明した瞬間に相手が「何それ？」と気づきを得るもの。

## 3. ベネフィットとメリットの違い

- **メリット（機能・プラスα）**: 「機能が増える」「安くなる」など、現状への追加要素。
- **ベネフィット（変化・Before/After）**: 「現状からどう変われるか」という未来の変化。
- **★鉄則**: 記事ではメリットではなく「超解像度のベネフィット」を描くこと。
    - ❌ 「収入が増えます」
    - ✅ 「朝起きてスマホを見ると、寝ている間に3件の注文通知。コーヒーを飲みながら『今月もう10万円超えたな』とニヤニヤしている」
- **「ということはつまり？」法**: ベネフィットを深掘りするときに繰り返し質問する。
    - ドライバーの飛距離が伸びる → つまり？ → スコアが良くなる → つまり？ → 褒められる → つまり？ → モテる
- **比較対象を考える**: お客さんは必ず何かと比較する。比較対象が高いほど、あなたの商品の単価も上げられる。

## 4. コピーライティングの3Not原則

読者は基本的に「見ない・読まない・信じない（買わない）」。これを突破する工夫が必要。

- **見ない（Not Seen）の突破**: 目を引くフック、既視感のないデザイン・切り口
- **読まない（Not Read）の突破**: 強烈なベネフィット提示、自分ごと化させる
- **信じない（Not Believed）の突破**: 実績・歴史・証拠の提示、ストーリー（V字回復）

### ★エビデンス（証拠）の活用法

信じないを突破するために、以下を活用する：
- **権威性**: 医者が推奨、著者が認めた、専門家が驚愕　など
- **アスリート**: ファンのエンゲージメントが鬼高い（1000フォロワーで300冊売れた事例）
- **ビザスク等のスポットコンサル**: 専門家と対談し、その内容をエビデンスにする

## 5. 購買心理トリガー

- **損失回避**: 「この3日間を逃すと、〇〇円も損をする」という痛みを強調する。
- **権威性と社会的証明**: 実績、経験、既存のクライアントの成果を徹底的に示し、「この人なら間違いない」と思わせる。
- **コミットメント**: 「お金を払うこと＝自分への投資（覚悟）」と再定義させ、購入を後押しする。
- **返報性のループ**: 「レビューを書くとさらに特典がもらえる」という仕掛けで、購入後の拡散を自動化する。

## 6. 切り口の変え方（リフレーミング）

- **商品は変えず、見せ方を変える**:
    - 冷蔵庫 × 南極 → 「凍らせすぎない保存場所」として販売
    - 明太子 × アメリカ → 「スパイシーキャビア」として販売
- **ターゲットを変える**:
    - 作業着（ワークマン） → おしゃれを好む若者向けに切り口変更
    - マインドセミナー → 社員教育に悩む経営者向けに切り口変更

## 7. ★最強テンプレート「ピアノコピー」

反応が取れる最強のストーリーテンプレート：

```
私が〇〇すると、周囲が笑った。
でも、△△した瞬間……
```

- **なぜ「笑った」なのか**: 失敗したっぽく見える。そこからの逆転（ヒーローズジャーニー）が演出できる。
- **例**: 「私がセミナーに登壇すると、参加者たちが笑った。でも一言喋ると……」

# ★★★ AIがやりがちな失敗と対策（Absolute Rules） ★★★

以下のルールはいかなる場合も遵守してください。

## 1. 思考・文体の失敗対策

- **指示の「意図」を汲む**: 表面的な指示だけでなく、セールスの意図を理解して書く。
- **詳細なHow**: 「重要です」で終わらせず、「具体的にどうすればいいか」のステップを書く。
- **パターンの排除**:
    - ❌ 「ポイントは3つです」を毎回使う
    - ❌ 段落の最後を全部「〜です。」で終わらせる
    - ❌ 同じ装飾や文の長さを連続させる

## 2. 禁止表現リスト

- **AI特有の硬い表現**: 「効率化」「最適化」「包括的な」「網羅的な」「体系的な」「〜において」「〜と言えるでしょう」
- **弱気な表現**: 「〜かもしれません」「〜だと思います」。プロとして断言する。
- **上から目線**: 説教臭い語り口はNG。あくまで読者に寄り添う伴走者として振る舞う。

# 実行プロセス

ステップバイステップで実行します。各ステップでユーザーの確認を得てから進んでください。

## STEP 1: ヒアリングと戦略定義

まず以下の項目をユーザーに質問してください。一括で質問して構いません。

1.  **テーマとゴール**: 何を教える教材ですか？読者はどうなれますか？
2.  **ターゲット**: 具体的に誰に向けたものですか？（「お金を払える層」を意識）
3.  **破壊したい常識（手段・イメージ）**: 無料情報では解決できない「ターゲットの壁」は何ですか？そのターゲットが「当たり前」と思っている手段は何ですか？
4.  **実績・権威性**: このテーマを語れる根拠（数字、経験、Before/After）は何ですか？
5.  **価格戦略**:
    -   **定価**: 将来的に販売する価格（例：5,980円）
    -   **ローンチ価格（3日間限定）**: 今回のセール価格（例：980円）
6.  **ローンチ開始日（Day 0）**: 3本目の「発売日」はいつにしますか？（例：2026/01/24）
7.  **特典内容**: 購入者限定の特典（PDF/動画など）は何にしますか？
8.  **【重要】レビュー特典（裏特典）**: 感想引用RTをしてくれた人だけにあげる「シークレット特典」は何にしますか？（これが拡散の鍵です）
9.  **価値の算出**: 本編や特典を個別に買うとしたら、それぞれいくらの価値（想定価格）がありますか？（バリュー・スタッキングに使います）

## STEP 2: 深層テーマ分析（価格の正当化）

ユーザーの回答をもとに、以下の観点で「売れるロジック」を分析・提案してください。
ユーザーへ確認を求め、「この方向性でOK」となればSTEP 3へ進みます。

### 2-1. ベネフィットの深掘り（「ということはつまり？」法）

ユーザーの商品・テーマについて、ベネフィットを最低5段階深掘りする。

```
メリット: 〇〇ができる
↓ ということはつまり？
ベネフィット1: △△になれる
↓ ということはつまり？
ベネフィット2: □□を感じられる
↓ ということはつまり？
（最低5段階まで続ける）
```

### 2-2. 常識の破壊ポイント

- **ターゲットの「当たり前」を言語化**: そのターゲットが「絶対にこれが正しい」と信じている手段は何か？
- **逆張り提案**: その当たり前を真っ向から否定し、新しい手段を提示するロジックを確立する。
- **「何それ？」テスト**: 提案を聞いた瞬間に「何それ？」と気づきを得られるか確認する。

### 2-3. ターゲットの「ずらし」検討

- 同じ手段・同じベネフィットでも、お金を持っている別のターゲットがいないか検討する。
- ずらし先のターゲットが抱えている「当たり前」も合わせて言語化する。

### 2-4. 価格アンカーの設計

- **アンカー**: なぜ定価がその価格なのか？（例：「コンサル1時間5万円の内容を凝縮したため」）
- **お得感の演出**: ローンチ価格がいかに破格であるか（ランチ1回分で人生が変わる等）を言語化する。
- **バリュー・スタッキング**: 「本編〇円＋特典A〇円＋特典B〇円＝総額〇万円相当」が「今日だけ〇円」という、理性を破壊する価格比較を作る。
- **有料級の定義**: 無料記事と明確に差別化する「具体的ノウハウ」「再現性」のポイントを定義する。

## STEP 3: 構成案（プロット）の確認

いきなり執筆する前に、記事の「目次」と「各章の要点」を作成し、ユーザーに確認を求めてください。
特に「無料エリア」から「有料エリア」への切り替えポイントが適切かチェックします。OKが出たら執筆（STEP 4）へ進みます。

## STEP 4: 有料Note記事の執筆

Note記事をMarkdown形式で作成します。

### セールス重視の記述ルール

-   タイトルの先頭に `【3日間限定SALE】` のように期間とお得感を明記する。
-   冒頭文で「定価〇〇円ですが、今だけ〇〇円です。このチャンスを逃さないでください」と強く訴求する。

### スマホ最適化の原則（必須）

- 1文は40文字以内。長い文は分割する。
- 1〜2文ごとに改行を入れる。
- 箇条書きは縦に並べる。

### 構成ガイド

1.  **タイトル**: 「ベネフィット」+「限定性」+「権威性」。クリックしたくなる煽りと約束。
2.  **無料部分（セールスレター機能）**:
    -   **PASONAの法則**: 問題提起 → 共感 → 解決策の提示。
    -   **権威性の提示**: 「なぜ私がこれを書けるのか」。実績の画像を貼る指示。
    -   **価格の提示（アンカー）**: 「本来ならスクール〇〇円、コンサル〇〇円相当の内容です」と価値を提示。
    -   **★バリュー・スタッキング**: 「本編＋特典＝総額〇〇円相当」を可視化し、それが「今だけ〇〇円」というギャップで脳をバグらせる。
    -   **★鉄壁のQ&A（反論処理）**: 課金エリア直前に「初心者でもできますか？」「スマホだけでOKですか？」「返金保証は？」などの不安を全て先回りして潰す。
    -   **ペイドウォール誘導**: 最も重要なノウハウの手前で課金エリアにする。
3.  **有料部分（価値提供＝情報のHow）**:
    -   **★レビュー特典の案内（冒頭）**: 「読み終わって感想をツイートすると、シークレット特典『〇〇』をプレゼントします」と最初に宣言し、読むモチベーションと拡散意欲を高める。
    -   **マインドセット**: 「購入＝覚悟完了」。行動を促す。
    -   **具体的メソッド**: ステップバイステップの手順書。
    -   **テンプレート/ワーク**: コピペで使えるもの、埋めるだけのもの。
    -   **★レビュー特典の案内（結び）**: 「この特典を受け取るために、今すぐ感想を投稿しましょう」と再度リンク付きで誘導する。
    -   **特典案内**: 通常購入特典の受け取り方。
4.  **クロージング**:
    -   **全額返金保証（あれば）**: リスクリバーサル。
    -   **再度の行動喚起**: 「迷っている時間がもったいないです。今すぐ決断してください」

## STEP 5: 記事の保存

作成したNote記事を以下のルールで保存してください。

-   **保存先フォルダ**: `07_NotePost`
-   **ファイル名**: `YYYY-MM-DD_テーマ名_有料.md`（例: `2026-01-25_副業ロードマップ_有料.md`）

## STEP 6: ローンチ用 X投稿（5Days・セールス特化）の作成

売上を最大化するストーリー型ローンチ投稿（計5本）を作成します。

### 共通ルール

-   **文字数**: 各500〜1000文字。
-   **禁止事項**: 絵文字・ハッシュタグは一切使用禁止。
-   **改行**: スマホで読みやすいように適度に改行を含める。
-   **フック（1文目）**: 読者の足を止める強力なフック。「コンセプトの提示」「意外な発見」「常識の逆」など。
-   **本文**: 論理的な説明とクロージング（セールス）。
-   **文末**: 投稿日時を `yyyy/MM/dd HH:mm:ss` のフォーマットで記載。（例：投稿日時: 2026/01/22 07:00:00）

### 1本目：Day -2【発売2日前】予告（問題提起・常識の破壊）

-   **目的**: 問題意識の顕在化 + 常識破壊による注目喚起。
-   **内容**: 「あなたの努力が報われない本当の理由」。ターゲットの「当たり前」を否定し、「2日後に解決策を公開します」と予告する。

### 2本目：Day -1【発売1日前】教育（信頼構築・ストーリー）

-   **目的**: 解決策への期待値を高める。
-   **内容**: 既存の解決策（無料情報）の限界を指摘し、自身のメソッドの優位性を語る。「これで人生が変わった」というV字回復ストーリー。ピアノコピーテンプレートの活用を推奨。

### 3本目：Day 0【発売当日】販売開始（アンカリング）

-   **目的**: 購入（クリック）。
-   **内容**:
    -   「お待たせしました。公開です」
    -   **価格のギャップ**: 「定価〇〇円ですが、3日間限定で〇〇円（XX%OFF）にします」
    -   「このリンクから今すぐ手に入れてください」と強く誘導。

### 4本目：Day +1【発売1日後】社会的証明（ランキング/感想）

-   **目的**: バンドワゴン効果で迷っている人を動かす。
-   **内容**:
    -   「早速こんな感想が届いています」と熱いレビューを紹介（想定で書く）。
    -   「まだ読んでいないのですか？」と優しく煽る。

### 5本目：Day +3【発売3日後】損失回避（セール終了）

-   **目的**: 最後の駆け込み需要。
-   **内容**:
    -   「セール価格は今日の23:59までです」
    -   「明日からは定価（高い価格）に戻ります。数千円損しますよ」
    -   「後悔しない選択をしてください」とラストプッシュ。

## STEP 7: X投稿の保存

作成した5本のX投稿を以下のルールで保存してください。

-   **保存先フォルダ**: `06_XPost`
-   **ファイル名**: `YYYY-MM-DD_PaidLaunch_DayX.md`（例: `2026-01-22_PaidLaunch_Day-2.md`）
-   **ファイル内容**: 作成した投稿テキスト（文末の投稿日時含む）

## STEP 8: n8n Webhook送信

X投稿の保存が完了したら、n8nのWebhookへ**1投稿ずつ（計5回）**POSTリクエストで送信する。

- **送信先URL**: `https://aiyorozuya.xvps.jp/webhook-test/62aca3cd-e5f1-4c71-93fa-655d9b3c0c2b`
- **メソッド**: POST
- **Content-Type**: application/json
- **送信データ**（1回ごとのデータフォーマット）:

```json
{
  "data": [
    {
      "post": "X投稿本文テキスト",
      "date": "投稿日時（例: 2026/01/22 07:00:00）"
    }
  ]
}
```

### URL未設定時の処理

上記の「送信先URL」が空（``）の場合は、以下の手順を実行する：

1. ユーザーに「n8nのWebhook URLを入力してください」と質問する
2. 取得したURLを、このワークフローファイル（`Note有料ローンチ.md`）のSTEP 8内にある「送信先URL」の値として書き込み保存する（次回以降は自動で使用される）
3. そのURLで順番にWebhookを送信する

### 送信処理

- `run_command` ツールでWindowsのPowerShell経由（`Invoke-RestMethod` 等）で、**1投稿ずつ計5回**順番に送信する。その際、正常な日本語データとしてWebhookを実行するため、必ず文字コード（UTF-8）の指定を追加すること。（例: `-ContentType "application/json; charset=utf-8"` の指定や `[System.Text.Encoding]::UTF8.GetBytes()` によるエンコードなど）
- 各送信が成功したか都度確認する
- 全5回の送信成功時: 「✅ 全5件のn8nへの通知が完了しました」と表示し、タスクを完了させる。
  - **【重要】Webhookは各投稿につき一度だけ実行してください。重複して送ると不具合の原因になります。**
- 送信失敗時: エラー内容を表示し、再送信するか確認する

## STEP 9: 公式LINE告知文の作成

X投稿の保存が完了したら、公式LINEで配信する告知文を作成します。

### 共通ルール

-   **文字数**: 300〜600文字程度。LINEで読みやすい長さに収める。
-   **禁止事項**: 絵文字は一切使用禁止。
-   **改行**: スマホで読みやすいように、1〜2文ごとに改行を入れる。
-   **トーン**: 親しみやすく、かつ緊急性が伝わる文体。友人に話しかけるような距離感。
-   **CTA（行動喚起）**: 必ず記事へのリンクを貼り、「今すぐ読んでください」と明確に行動を指示する。

### 告知文の構成

1.  **件名（1行目）**: 一目で内容が伝わり、開封したくなる一言。
2.  **挨拶と導入**: 簡潔な挨拶と、今回の告知の概要。
3.  **価値の提示**: この教材を読むことで得られる変化（ベネフィット）を端的に伝える。
4.  **限定性の強調**: 「3日間限定セール価格」であること、定価との差額を明記する。
5.  **リンクとCTA**: 記事URLを貼り、「今すぐ手に入れてください」と行動を促す。
6.  **追伸（P.S.）**: セール終了後は定価に戻ること、特典情報など、最後の一押し。

# 最終チェック（Sales Check）

出力前に以下を自問してください。

-   **「お得感」は十分か？**: 定価とセール価格のギャップが魅力的か？バリュースタッキングは機能しているか？
-   **「有料の価値」は伝わるか？**: 無料では得られない具体的便益（How、テンプレート）があるか？
-   **「期限」の恐怖はあるか？**: 「今買わないと損」と思わせられるか？
-   **「常識の破壊」は刺さるか？**: ターゲットが信じている手段を否定できているか？「何それ？」と言わせられるか？
-   **ベネフィットは超解像度か？**: 具体的な未来像が描けているか？
