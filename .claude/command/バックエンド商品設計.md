# バックエンド商品設計スキル

このスキルは、オンライン講座・コンサルティング・コミュニティなどのバックエンド商品を「売れて満足度が高い」形に設計します。
LPやウェビナー台本など販売ツールの作成は別ワークフロー（`/LP作成`、`/ウェビナー台本作成`）に任せ、**商品そのものの設計**に特化します。

## 使用方法

1. Phase 1: ヒアリング（商品の中身・実績・ターゲット・価格帯の収集）
2. Phase 2: ベネフィット深掘り（10個書き出し + 3層構造で深掘り）
3. Phase 3: 市場リサーチ（ターゲットの「当たり前」と競合のリサーチ）
4. Phase 4: コンセプト設計（常識の破壊 → パッケージ設計 → ターゲットのずらし）
5. Phase 5: オファー設計（価格・特典・保証・Value Stack・希少性/緊急性）
6. Phase 6: コンテンツ構成設計（カリキュラム・サポート内容・提供形式）
7. Phase 7: 商品設計書の出力（Markdownファイルとして保存）

# ★★★ 知識ベース（Reference Knowledge） ★★★

以下の知識を全ての出力に適用し、「売れて満足度が高い」バックエンド商品に仕上げてください。

## 1. ★最最重要★ 無形ビジネスの本質（教えるな、体験させろ）

無形ビジネス（コンテンツビジネス）の根本的な間違い＝「教えること」が目的だと思っていること。

```
【大原則】
├─ ❌ ノウハウを教える → 講師のエゴ。お客さんは動かない
├─ ✅ 「何を体験させるか」「何をやらせれば人生が変わるか」で考える
└─ ✅ お客さんが自走する仕組みを作る → スキルとして身につけさせる

★ノウハウではなく「気づき」が必要。「体験」と「自走の仕組み」が価値を決める★
```

### 体験ベースのコンテンツ設計

```
【正しい手順】
1. まず1対1（モニター）で体験を提供する
2. 気づき・スキルのフィードバックを受け、成功パターンを体系化する
3. 「なぜこの体験が重要か」を気づかせるカリキュラムに落とし込む
4. カリキュラムが固まってから集客を始める

【体験の例】
├─ 野球→まず振らせる（理論は後）/ 水泳→まず水に入れる（説明は後）
└─ 集客→集客で人生が変わると気づかせる（手法は後）

★情報で戦うとネットに負ける。体験で戦え。リアル体験を入れろ★
```

## 2. ★最重要★ PMF（プロダクトマーケットフィット）

商品設計で最も重要な考え方。「自分が作りたいものを作る」のではなく、「市場が求めているものに商品を合わせる」。

```
【PMFの定義】
├─ Product（商品）をMarket（市場）にFit（適合）させること
└─ 営業不要で向こうから「売ってください」と言われる状態を作ること

【よくある失敗（プロダクトアウト）】
├─ ❌ 自分の経験をそのまま商品化 / 自分ができる範囲で作る
├─ ❌ 「たった1人のペルソナ」に絞りすぎて市場がない
└─ ❌ 営業力で売ろうとする（=PMFできていない証拠）

★営業が必要な商品はPMFできていない★
```

### バリュープロポジション（Value Proposition）

```
★狙うべきポジション = 競合ができない × 自社ができる × 顧客にニーズがある★
├─ ❌ 競合と被る領域で戦う → 価格競争に陥る
└─ ✅ 競合が入れない独自の価値を提供する → 営業不要で売れる
```

### ★リサーチの落とし穴★

```
❌ 顧客に「何が欲しいか」を聞くな → 潜在ニーズに気づいていないので浅い悩みしか返らない
✅ 「今売れている・集まっている・バズっている商品」を調べ、本質的価値を分析→横展開

【PMF事例】
├─ 何もできないスマホ（9億円調達）→ 潜在ニーズ: デジタルデトックス
├─ YouTube集客→YouTubeプロモーター講座 → 市場成熟に合わせ再フィット
└─ インスタ講座溢れる中でスポークスマン戦略 → 独自のバリュープロポジション

★中身は変えなくていい。見せ方（パッケージ）を市場に合わせろ★
```

### コンテンツ小分け戦略

```
高単価オールインワンが溢れ、選択が重い市場 → 逆張りで小分けにする
例: インスタ講座40万→リール講座9,800円+ストーリー講座5万+… → LTV同じ
★機能を増やすより、研ぎ澄ましてシンプルにする方が刺さることもある★
```

## 3. コンセプト設計（パズル思考）

コンセプトメイキングはセンスではなくスキル。「真新しいもの」を作る必要はない。既存の要素を掛け合わせた「目新しいもの」を作る。

### ベネフィット ≠ メリット

```
メリット：プラスアルファの要素（機能が追加される）
ベネフィット：ビフォーアフターの変化（現状からどう変われるか）

★コンセプト作りにはベネフィットの提示が必須★
```

### パッケージ戦略（中身は変えず見せ方を変える）

```
商品の中身→変えなくてよい / パッケージ（見せ方・切り口）→ここを変える
例：ただの水→リキッドデス（10倍価格） / 不動産営業マン募集→週末不動産副業（殺到）
```

### ★常識の破壊★（コンセプトの核心）

```
【やること】
1. ターゲットが「良かれと思ってやっていること（手段・イメージ・常識）」を特定する
2. その常識が実は間違っている（問題である）ことを示す
3. より優れた手段（あなたの商品）を提示する

【例】
├─ 常識：集客にはSNS毎日投稿が必須 → 破壊：SNS不要、セミナー×広告で自動集客
└─ 常識：副業は動画編集で月5万が限界 → 破壊：週末不動産だけで月100万も可能

★良いコンセプトは「え、何それ？」と気づきを与えるもの★
```

### ターゲットのずらし

```
【考え方】
1. 手段を具体的に書き出す
2. その手段にまだ気づいていない別のターゲットを探す
3. お金を持っているターゲットに寄せる

【大きな悩みカテゴリ】
├─ お金（稼ぐ・守る）
├─ ビジネス（集客・売上・効率化）
├─ 自己成長（スキル・キャリア）
├─ 恋愛（モテ・パートナーシップ）
├─ 健康（体調・美容・メンタル）
└─ 自己実現（自由・影響力・承認）

★同じ手段・同じベネフィットでも、ターゲットをずらすだけで新市場が生まれる★
```

## 4. ベネフィットの3層構造

「時間が節約できます」「お金が稼げます」で止まるな。深掘りせよ。

```
第1層（機能的ベネフィット）
├─ 何ができるようになるか？
│
第2層（感情的ベネフィット）
├─ その結果、どんな感情になるか？
│
第3層（自己実現ベネフィット）
└─ 周りの目はどう変わるか？人生はどう変わるか？
```

**★必ず第3層まで掘り下げる★**

## 5. 3つの反論処理（3 Beliefs / Objections）

バックエンド商品では、顧客が持つ3つの壁を先回りして設計に組み込む。

| 壁 | 顧客の心の声 | 設計への反映 |
|----|-------------|-------------|
| **Vehicle（方法）** | 「本当にそのやり方でうまくいくの？」 | 成功事例・社会的証明をカリキュラムの冒頭に配置 |
| **Internal（自分）** | 「私にもできるの？」 | 初心者向けサポート・段階的ステップ設計 |
| **External（環境）** | 「時間がない/お金がないけど大丈夫？」 | 時間対効果のリフレーミング・分割払い・ROIの提示 |

## 6. 価格設計の心理学

### アンカリング効果

最初に高い価格（得られる価値・比較対象）を見せてから、実際の価格を見せると安く感じる。

### Value Stack（価値の積み上げ）

```
【含まれるもの】
✅ 本編カリキュラム（〇〇万円相当）
✅ 特典1: テンプレート/ツール（〇万円相当）
✅ 特典2: 個別サポート（〇万円相当）
✅ 特典3: コミュニティ参加権（〇万円相当）
───────────────
合計〇〇万円相当 → 特別価格〇〇万円
```

### リスクリバーサル（リスクゼロ化）

- 返金保証、成果保証、無期限サポートなど
- 「買わない理由」をすべて潰す

## 7. ブランディングの鉄則

- **マイナス要素は先に自分から開示せよ。** 後から発覚すると信頼崩壊。「向いていない人」を明記すると逆に信頼UP。
- **「どう思われたいか」を決め**、商品名・言葉選び・特典を一貫させる。

# ★★★ AIがやりがちな失敗と対策 ★★★

## 失敗1: ベネフィットが浅い

- **現象**: 「売上が上がります」で止まる → ★3層構造で第3層（自己実現）まで到達する★

## 失敗2: ターゲットが広すぎる / 狭すぎる

- **現象**: 「起業したい人向け」と大雑把 or 「たった1人のペルソナ」に絞りすぎて市場がない
- **対策**: ★まず市場があるか確認し、その上で悩み・問題・ターゲット・地域で絞る★
- **意識**: 市場規模とニッチさのバランスを取る（PMFの視点）

## 失敗3: 競合と同じことを言う

- **現象**: 「誰でも簡単」「最強メソッド」など → ★常識の破壊ポイントを必ず1つ以上入れる★

## 失敗4: オファーが弱い

- **現象**: 商品の中身だけ説明して、なぜ「今」買うべきかが不明
- **対策**: ★希少性（物理的制限）と緊急性（論理的理由）を必ず設計する★

## 失敗5: プロダクトアウトで作る

- **現象**: 自分の経験・スキルをそのまま商品化し、市場にニーズがない
- **対策**: ★PMFの視点で「今、市場で売れているもの」からリサーチし、商品を市場にフィットさせる★
- **確認**: 営業しないと売れない商品→PMFできていない証拠

## 失敗6: ノウハウを教えるだけのカリキュラム

- **現象**: 情報の羅列で顧客が動かない。「何からやればいいか分からない」状態
- **対策**: ★体験ベースのカリキュラムにする。「何をやらせれば気づくか」から逆算して設計する★
- **確認**: カリキュラムに体験・ワーク・アウトプットが含まれているか？

## 禁止表現

- ❌ 「効率化」「最適化」「包括的な」「網羅的な」「体系的な」
- ❌ 「〜することが可能です」→「〜できます」に
- ❌ 「ソリューション」「パラダイムシフト」などカタカナ語の乱用
- ❌ 架空の数字や実績は絶対NG

```markdown
□ グループコンサルティング
□ 個別コンサルティング（1on1）
□ コミュニティ（月額制）
□ 合宿・ブートキャンプ
□ その他（具体的に）

**【一言で言うと】**

---

### 2. ターゲット

**【年齢層・属性】**
（例：30代の副業をしたい会社員、すでに月商100万の個人事業主など）

**【今の状態（Before）】**
ターゲットは今、どんな状態で、何に困っていますか？
具体的に書いてください（数字があれば最高）。

**【理想の状態（After）】**
この商品を受けた後、どうなっていてほしいですか？
具体的な成果を書いてください（数字があれば最高）。

**【ターゲットが「良かれと思って」やっている手段（★重要）】**
ターゲットが目標達成のために「これが正しい」と信じてやっていることは何ですか？

顧客に「何をやらせれば」人生が変わるきっかけになると思いますか？
ノウハウではなく、体験・ワーク・アウトプットの観点で考えてください。
（例：「実際にn8nを触って1つのワークフローを完成させる」「初めてのウェビナーを開催する」）

**【提供方法】**
□ 動画教材（録画済み）

□ 個別面談
□ チャットサポート（LINE・Discord等）
□ テンプレート/ツール提供
□ コミュニティ（Slack・Facebook等）
□ リアル合宿・ワークショップ
□ その他
```

## Phase 2: ベネフィット深掘り

ヒアリング結果を基に、以下のフォーマットでベネフィットを10個以上書き出し、それぞれ3層構造で深掘りする。

```markdown
## ベネフィット分析

### ベネフィット1: [タイトル]
- 第1層（機能的）: [何ができるようになるか]
- 第2層（感情的）: [どんな気持ちになるか]
- 第3層（自己実現）: [周囲からどう見られるか/人生がどう変わるか]

### ベネフィット2: ...
（以下、最低10個）
```

書き出したベネフィットの中から、**最もギャップが大きく、ターゲットの目を引くもの**をトップ3として選定し、ユーザーに確認を求める。

## Phase 3: 市場リサーチ（★必ずWeb検索を実行★）

### ★PMFリサーチ（最優先）★

```
【やるべきこと】
★顧客に「何が欲しいか」を聞くのではなく、以下をリサーチする★

1. 今、同じジャンルで「売れている」商品は何か？
2. 今、同じジャンルで「集まっている」商品は何か？
3. 今、同じジャンルで「バズっている」コンテンツは何か？
4. それらが売れている「本質的な価値（潜在的ニーズ）」は何か？
5. その価値を自分の商品に横展開できないか？

【検索クエリ例】
├─ 「[テーマ] 講座 人気」「[テーマ] スクール 評判」
├─ 「[テーマ] セミナー 満席」「[テーマ] 講座 期待」
└─ TikTok・YouTube・Xで「[テーマ]」のバズっている投稿を調査

★潜在的ニーズを掘れ。顕在的な悩みは浅い。顧客も気づいていない欲望を見つけろ★
```

### ターゲットの「当たり前」リサーチ

```
【リサーチ先】
├─ Yahoo!知恵袋
├─ X（Twitter）
├─ YouTubeのコメント欄
├─ Google検索「〇〇 失敗」「〇〇 やめた」「〇〇 意味ない」
└─ 競合のLP・セールスページのレビュー

【検索クエリ例】
├─ 「[テーマ] 挫折 site:chiebukuro.yahoo.co.jp」
├─ 「[テーマ] 高額 後悔」「[テーマ] コンサル 意味ない」
└─ 「[テーマ] 独学 限界」

【収集すべきもの】
├─ ターゲットが「当たり前」だと思っている手段（常識）
├─ その手段でうまくいかなかった具体的な失敗談
├─ 高額商品への不安・不満の声
└─ 「こういうのがあったらいいのに」という潜在ニーズ
```

### 競合分析（バリュープロポジション特定）

```
【リサーチすべきもの】
├─ 競合の価格帯・提供内容・特典
├─ 競合が使い倒しているクリシェ（使い古されたフレーズ）
├─ 競合の弱点・顧客の不満
├─ 競合が「機能の戦い」に陥っているポイント
├─ 同じ手段で高単価で売れている異業種の事例
└─ ★競合が入っていない × 自社ができる × 顧客にニーズがある領域★
```

## Phase 4: コンセプト設計

リサーチ結果を元に、以下のフォーマットでコンセプトを設計し、ユーザーに確認を求める。

```markdown
## 商品コンセプト定義
```

## Phase 6: コンテンツ構成設計

商品の中身（カリキュラム・サポート内容）を設計する。

### ★カリキュラム設計の大原則（体験ベース）★

```
【最重要】教えるカリキュラムではなく「体験するカリキュラム」を作れ

❌ ノウハウを教える → お客さんはメモを取るだけで動かない
✅ 体験させて気づかせる → お客さんが自走してスキルを身につける

★「この体験がいかに重要か」を気づかせ → 体験させ → スキルにする★
```

### 各章の構造

```
1. この章のゴール（何ができるようになるか）
2. Why（なぜこの体験/スキルが重要か — 気づきを与える）
3. What（概念の説明 — 最小限でOK）
4. ★体験・ワーク（実際にやらせる — ここがメイン）★
5. 振り返り（何に気づいたか？どんなスキルが身についたか？）
6. 事例（実体験・クライアント事例 — 「あの人もこの体験で変わった」）
7. 章のまとめ + 【今日やること】（1つだけ、すぐできること）

【意識すべきこと】
├─ 各章に必ず1つ以上の「体験・ワーク・アウトプット」を入れる
├─ 各章を終えるごとに「小さな成功体験」を得られる設計にする
├─ 情報を詰め込みすぎない（1章1テーマ）
├─ 体験→フィードバック→改善のサイクルを組み込む
├─ 難しい理論は体験の後に教える（先に振らせてから打ち方を教える）
└─ 「こんな人には向いていません」を明記して信頼を上げる
```

### サポート体制の設計

```
【検討すべきサポート要素】
├─ 質問対応: チャット / メール / Zoom面談
├─ 添削・フィードバック: 課題提出 → 添削
├─ コミュニティ: Slack / Discord / Facebookグループ
├─ ライブ講義: 週1 / 月2 / 月1
├─ 個別面談: 月1回 / 期間中〇回
└─ アップデート保証: コンテンツの追加・更新
```

# 最終チェック

出力前に以下を自問してください。

- **PMFできているか？**: 営業しなくても「欲しい」と言われる商品か？市場にフィットしているか？
- **バリュープロポジションは明確か？**: 競合ができない × 自社ができる × 顧客にニーズがある領域を取っているか？
- **体験ベースのカリキュラムか？**: 「何を教えるか」ではなく「何を体験させるか」で設計しているか？
- **顧客が自走する仕組みがあるか？**: ノウハウの羅列ではなく、体験→気づき→スキルの流れがあるか？
- **ベネフィットは第3層まで掘り下げたか？**: 機能→感情→自己実現の変化が明確か？
- **常識の破壊は入っているか？**: ターゲットの「当たり前」を覆す切り口があるか？
- **ターゲットのずらしを検討したか？**: お金を持っている層にリーチできているか？
- **オファーは魅力的か？**: Value Stackで「これだけついてこの価格！？」と思えるか？
- **反論処理は設計に組み込まれているか？**: 方法・自分・環境の3つの壁を潰しているか？
- **希少性・緊急性は論理的か？**: 「なぜ今」「なぜ限定」が物理的要因で説明されているか？
- **マイナスを先出ししたか？**: 「向いていない人」を正直に書いているか？
- **架空の実績を使っていないか？**: 全ての数字・事例はユーザーから提供されたものか？
