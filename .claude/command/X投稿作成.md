# X投稿作成スキル

このスキルは、ナレッジベースの情報を活用し、高品質なX（旧Twitter）用の長文ポストを作成します。

## 使用方法

1. STEP 1: テーマの確認
2. STEP 2: 情報源の特定
3. STEP 3: 文体の決定
4. STEP 4: 投稿日時の指定
5. STEP 5: 投稿のドラフト作成
6. STEP 6: ユーザー確認
7. STEP 7: ファイル保存
8. STEP 8: n8n Webhook送信

# ★★★ 知識ベース（Reference Knowledge） ★★★

## 1. フックの法則（1文目で勝負が決まる）

X投稿の成否は**1文目**で決まる。以下のパターンを活用せよ。

| パターン | 説明 | 例 |
|---------|------|-----|
| **常識の否定** | 多くの人の「当たり前」を覆す | 「毎日投稿しても伸びないのは当たり前」 |
| **意外な発見** | 知的好奇心を刺激する | 「AIに仕事を奪われる職種の特徴を調べたら意外な共通点があった」 |
| **コンセプト提示** | 新しい概念を提示する | 「"怠惰な成功者"という人種がいる」 |
| **問いかけ** | 読者に考えさせる | 「なぜ頭のいい人ほど行動が遅いのか？」 |
| **数字のインパクト** | 具体的な数字で引きつける | 「1年で売上が3倍になった理由は"やめたこと"にあった」 |

**★禁止★**: 説明から入る冒頭（「今回は〇〇について解説します」）

## 2. 本文の構成法則

```
【基本構造】
1文目（フック）
  ↓
なぜそう言えるのか（論理的説明）
  ↓
具体例・エピソード
  ↓
読者へのメッセージ（気づき・示唆）
```

### 納得感を生む要素

- **論理の展開**: 主張→理由→具体例の流れ
- **自分の体験**: 「俺も昔は〜だった」「実際にやってみたら〜」
- **第三者の事例**: 「〇〇さんの話を聞いたら〜」
- **数字の裏付け**: 「〇〇%の人が〜」

## 3. スマホ最適化の鉄則

- **1文を短く**: 目安30文字以内
- **頻繁に改行**: 2〜3文ごとに空行
- **箇条書きを活用**: 列挙は縦に並べる
- **句読点より改行**: 「。」の後は基本的に改行

## 4. AIっぽさを消すテクニック

### 禁止表現

- ❌ 絵文字・ハッシュタグ
- ❌ 「〜することが可能です」
- ❌ 「〜において」「〜における」
- ❌ 「包括的な」「網羅的な」「体系的な」
- ❌ 「いかがでしたか？」
- ❌ 完璧すぎる文構造

### 推奨表現

- ✅ 短い断言（「〜だ」「〜しかない」）
- ✅ 口語的な表現（「ぶっちゃけ」「正直」）
- ✅ 感情の吐露（「これマジで驚いた」）
- ✅ 留保の挿入（「もちろん例外はある」）

# ★★★ AIがやりがちな失敗と対策 ★★★

## 失敗1: フックが弱い

- **現象**: 説明から入る、当たり前のことを言う
- **対策**: ★「常識の否定」か「意外な発見」から始める★

## 失敗2: 論理的すぎて読みにくい

- **現象**: 接続詞が多い、文が長い、改行が少ない
- **対策**: ★短文で断言し、頻繁に改行する★

## 失敗3: 文体サンプルを無視する

- **現象**: 指定された文体と違う一人称・語尾を使う
- **対策**: ★サンプルを分析し、その特徴を完全に模倣する★

## 失敗4: パターン化する

- **現象**: 同じ構成、同じ文の長さ、同じ語尾の連続
- **対策**: ★意図的に不規則にする★

# 実行プロセス

**各ステップごとにユーザからの入力を待ち、全ての情報が揃ってから次のステップに進んでください。一度に全ての質問をしないでください。**

## STEP 1: テーマの確認

ユーザに質問してください：

```
**ネタにしたいテーマは何ですか？**
（例：AI活用、マーケティング、生産性向上、マインドセットなど）
```

## STEP 2: 情報源の特定

ユーザに質問してください：

```
**どの情報源を使いますか？**
（フォルダ名、ファイル名、または「特に指定なし」）
```

指定されたフォルダやファイルをナレッジベースから検索し、テーマに関連する情報を読み込む。

## STEP 3: 文体の決定

ユーザに質問してください：

```
**どの文章を参考にして文体を作りますか？**
（参考にするファイル名、投稿者名、またはURLを貼り付け、または「おまかせ」）
```

指定があればその文体を分析し、以下を箇条書きでまとめる：

```
【文体分析】
- 一人称: 私 / 僕 / 俺 / なし
- 語尾: だ・である / です・ます / 〜だよね
- 改行頻度: 多い / 適度 / 少ない
- 口癖・特徴的な表現: 
- 感情の出し方: 冷静 / 熱血 / 論理的
```

## STEP 4: 投稿日時の指定

ユーザに質問してください：

```
**投稿日時はいつにしますか？**
（例：2026/02/10 07:00:00、または「指定なし」）
```

## STEP 5: 投稿のドラフト作成

収集した情報（テーマ、情報源、文体、投稿日時）をもとに、以下のガイドラインに従ってドラフトを作成する。

### 投稿ガイドライン

| 項目 | ルール |
|-----|--------|
| **文字数** | 必ず500文字以上 |
| **絵文字** | 一切使用しない |
| **ハッシュタグ** | 使用しない |
| **1文目（フック）** | 「常識の否定」「意外な発見」「コンセプト提示」のいずれかで読者の足を止める |
| **本文** | 論理的な説明と具体例で納得感を生む |
| **改行** | スマホで読みやすいよう頻繁に改行 |
| **文末** | 投稿日時を `yyyy/MM/dd HH:mm:ss` のフォーマットで記入（例：投稿日時: 2026/02/10 07:00:00）|

## STEP 6: ユーザー確認

作成したドラフトを提示し、ユーザーに確認を求める。
（ドラフトはファイルとして保存せず、チャット上でのみ提示してください）

## STEP 7: ファイル保存

ユーザーが「OK」または「承認」を出した場合のみ、以下の処理を行う。

- **保存先フォルダ**: `06_XPost`
- **ファイル名**: `YYYY-MM-DD_テーマ名.md`（例: `2026-02-10_AIマーケティング.md`）

## STEP 8: n8n Webhook送信

ファイル保存が完了したら、n8nのWebhookへPOSTリクエストを送信する。
**（注意：送信するX投稿の本文は、マークダウン装飾をすべて外したプレーンテキストにしてください）**

- **送信先URL**: `https://aiyorozuya.xvps.jp/webhook/62aca3cd-e5f1-4c71-93fa-655d9b3c0c2b`
- **メソッド**: POST
- **Content-Type**: application/json
- **送信データ**:

```json
{
  "data": [
    {
      "post": "作成したX投稿の本文テキスト（マークダウンなしのプレーンテキスト）",
      "date": "投稿日時（STEP 4で指定された日時。例: 2026/02/10 07:00:00）"
    }
  ]
}
```

### URL未設定時の処理

上記の「送信先URL」が空（``）の場合は、以下の手順を実行する：

1. ユーザーに「n8nのWebhook URLを入力してください」と質問する
2. 取得したURLを、このワークフローファイル（`X投稿作成.md`）のSTEP 8内にある「送信先URL」の値として書き込み保存する（次回以降は自動で使用される）
3. そのURLでWebhookを送信する

### 送信処理

- `run_command` ツールでWindowsのPowerShell経由（`Invoke-RestMethod` 等）で送信する。その際、正常な日本語データとしてWebhookを実行するため、必ず文字コード（UTF-8）の指定を追加すること。（例: `-ContentType "application/json; charset=utf-8"` の指定や `[System.Text.Encoding]::UTF8.GetBytes()` によるエンコードなど）
- 送信成功時: 「✅ n8nへの通知が完了しました」と表示し、タスクを完了させる。
  - **【重要】Webhookは一度だけ実行し、成功の場合はタスクを完了させてください。2回送ると不具合の原因になります。**
- 送信失敗時: エラー内容を表示し、再送信するか確認する

# 最終チェック

出力前に以下を自問してください：

- **フックは強力か？**: 1文目で読者の足を止められるか？
- **納得感はあるか？**: 論理と具体例で説得力があるか？
- **スマホで読みやすいか？**: 短文・頻繁な改行になっているか？
- **文体は一致しているか？**: 指定されたサンプルと同じ一人称・語尾か？
- **500文字以上あるか？**
- **絵文字・ハッシュタグを使っていないか？**
