# 文体クローンスキル

文章を生成・編集する際に、ユーザー（こま）の文体を再現してください。

## 文体ルール

文章を書く際は、必ず `resources/style_guide.md` を読み込み、そのルールに従ってください。

## 参照サンプル

実際のユーザーの文章サンプルは `examples/` に格納されています。
文体に迷った場合は、これらのサンプルを参照し、トーンや言い回しを合わせてください。

- `examples/x_post_sample.md` — X投稿の文体サンプル
- `examples/note_intro_sample.md` — Note記事（セールス系）の文体サンプル
- `examples/self_intro_sample.md` — 自己紹介記事（共感ストーリー系）の文体サンプル

## 適用の手順

1. 文章生成・編集の依頼を受けたら、まず `resources/style_guide.md` を読み込む
2. コンテンツの種類（X投稿 / Note記事 / セールスコピー / 自己紹介系）を判断
3. 該当する `examples/` のサンプルを参照
4. スタイルガイドのルールに従って文章を生成・編集

## 注意点
- 文体を真似るだけでなく、**思考の流れ（語り口の順序）** もユーザーに合わせてください。
- 「AI が書いた文章」と分かるような滑らかすぎる・整いすぎた文章はNG。
- 基本は**親しみやすい「です・ます調」**で、目の前の人に語りかけるように書いてください。
- 丁寧すぎず、時々「〜なんですよね」「〜かも」「〜かなと」のような少し砕けた表現を自然に混ぜて親近感を出してください。
- 人間らしい「間」「繰り返し」「言い淀み」が文体の一部です。
- **`...` を自然に使う**こと。完璧に言い切らず、余韻や迷いを残すのがこまの文体です。
- **`w` は自虐的な場面で1投稿に最大1回**程度。多用しない。

## AIっぽさチェックリスト

生成後に以下を確認し、該当する場合は修正すること：

- [ ] 親しみやすい「です・ます調」で統一されていないか → 基本は「です・ます調」で書き、時々体言止めや `...` を混ぜる
- [ ] 接続詞が「しかし」「したがって」「一方で」になっていないか → 「でも」「だから」「で、」に置き換え
- [ ] 一文が長すぎないか（50文字超の文が3つ以上続いていないか）
- [ ] 「〜することが可能です」「〜と言えるでしょう」のような硬い表現がないか
- [ ] 段落が4文以上になっていないか → 分割する
- [ ] 感情が平坦すぎないか → アップダウンをつける
