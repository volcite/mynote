# n8nワークフローバリデータースキル

n8nワークフローのJSONファイルを検証し、潜在的な問題を検出してください。

## 検証手順

1. **自動バリデーションの実行**
   - 対象がJSONファイルの場合、以下のスクリプトを実行してください。
   - コマンド: `python scripts/n8n_validator.py <ファイルパス>`
   - チャットに直接貼り付けられた場合は、目視で構造を確認してください。

2. **ベストプラクティスとの照合**
   - `resources/n8n_best_practices.md` を読み込み、ワークフローがベストプラクティスに沿っているか確認してください。

3. **検証レポートの作成**
   - `examples/validation_output_example.md` のフォーマットを参考に、以下の構成で回答してください：
     1. **ワークフロー概要**: ノード数、接続数、使用ノードタイプ
     2. **構造チェック結果**: 必須フィールド、接続の整合性
     3. **セキュリティチェック**: Credential、Webhook設定
     4. **改善提案**: エラーハンドリング、命名規則等
     5. **総合判定**: 本番利用可能 / 要修正 / 致命的問題あり

## 注意点
- JSON構文エラーがある場合は、まずその修正を提案してください。
- ノード間の接続が切れている箇所は最優先で報告してください。
- Credential情報が直接埋め込まれている場合は、セキュリティリスクとして警告してください。
